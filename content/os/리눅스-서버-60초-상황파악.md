> Netflix [Linux performance analysis in 60,000ms](https://medium.com/netflix-techblog/linux-performance-analysis-in-60-000-milliseconds-accc10403c55) 요약. `sysstat` 패키지 필요.

## 60초 체크리스트

```bash
uptime
dmesg | tail
vmstat 1
mpstat -P ALL 1
pidstat 1
iostat -xz 1
free -m
sar -n DEV 1
sar -n TCP,ETCP 1
top
```

USE Method: Utilization / Saturation / Error (CPU·mem·disk).

| 명령 | 볼 것 |
| --- | --- |
| uptime | load 1/5/15분 추세 |
| dmesg | OOM, TCP drop |
| vmstat | `r`>CPU수, `si/so`≠0, us/sy/wa |
| mpstat | CPU별 불균형(싱글스레드) |
| pidstat | 프로세스별 %CPU |
| iostat | r/s·w/s, await |
| free -m | buffers/cached (0 근접 주의) |
| sar -n DEV | rx/tx KB/s |
| sar -n TCP | active/passive, retrans |
| top | 종합 (패턴 추적용은 스크롤 출력 명령이 나음) |
