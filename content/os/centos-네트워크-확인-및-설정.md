## 현재 네트워크 정보 확인

- `ifconfig` : 인터페이스별 IP주소, Subnet Mask, Broadcast 주소, MAC 주소 등 정보 출력
  ```bash
$ ifconfig
eth0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 192.168.1.202  netmask 255.255.255.0  broadcast 192.168.1.255
        inet6 fe80::d028:99ae:4f92:2224  prefixlen 64  scopeid 0x20<link>
        ether bc:5f:f4:c6:8c:e5  txqueuelen 1000  (Ethernet)
        RX packets 494780588  bytes 44544371072 (41.4 GiB)
        RX errors 0  dropped 61940  overruns 0  frame 0
        TX packets 4393602710  bytes 6633189402791 (6.0 TiB)
        TX errors 0  dropped 0 overruns 0  carrier 0  collisions 0
  ```

- `/etc/sysconfig/network-scripts/ifcfg-eth0` : 인터페이스별 네트워크 정보 조회
  - 장치명은 시스템에 따라, eth0, eth1, encap1, encap2 등 다양한 형태로 표시될 수 있으니, `ifconfig`로 네트워크 인터페이스 이름을 확인

  ```bash
$ cat /etc/sysconfig/network-scripts/ifcfg-eth0
TYPE=Ethernet
PROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=none
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_FAILURE_FATAL=no
IPV6_ADDR_GEN_MODE=stable-privacy
NAME=enp3s0
UUID=54b82fed-8531-45d4-82a8-969bbd4ee110
DEVICE=enp3s0
ONBOOT=yes
DNS1=168.126.63.1
IPADDR=192.168.1.202
PREFIX=24
GATEWAY=192.168.1.254
  ```

## 네트워크 설정 변경

- `ifconfig` : Config 명령어로 IP address, subnet mask, broadcast 주소를 설정다.
  - CLI 모드 상태에서 ifconfig 명령으로 IP 주소를 설정하면, 시스템 메모리 상에 저장되어 동작하지만, 휘발성 상태이기 때문에 서버 재시작을 하면 내용은 사라짐. 서버 재시작 후에도 영구적으로 설정 변경하려면 아래와 같이 설정 파일을 이용하여 설정해야 함.
  - \[NIC 장치명\] = eth0
  - \[IP 주소\] = 101.102.103.75
  - \[서브넷마스크 주소\] = 255.255.255.128
  - \[브로드캐스트주소\] =101.102.103.127
  - up = up 또는 down 선택 가능

  ```bash
ifconfig [NIC 장치명] [IP 주소] netmask [서브넷마스크 주소] broadcast [브로드캐스트주소] up

# (예)
ifconfig eth0 101.102.103.75 netmask 255.255.255.128 broadcast 101.102.103.127 up
  ```
- `/etc/sysconfig/network` : Hostname 변경, 네트워크 활성화(yes/no)를 위한 설정
  - NETWORKING=yes : 네트워크를 활성화(yes), 비활성화(no)를 설정
  - NETWORKING\_IPV6=no : IPv6 통신 지원을 yes/no 로 설정
  - HOSTNAME=MYSERVER : 시스템의 hostname을 설정

  ```bash
$ cat /etc/sysconfig/network
NETWORKING=yes
NETWORKING_IPV6=no
HOSTNAME=myserver
GATEWAY=101.102.103.1
  ```

- `/etc/resolv.conf` : DNS 서버 설정
  - 주요 인터넷 사업자별 DNS 대표 주소
    - **SKB** : 기본 : 221.139.13.130, 보조 : 210.94.0.73
    - **KT** : 기본 : 168.126.63.1,  보조 : 168.126.63.2
    - **LG U+  : **기본 : 164.124.101.2, 보조 : 203.248.252.2
    - **Google** : 기본 : 8.8.8.8, 보조 : 8.8.4.4
    - **클라우드 플레어** : 기본 : 1.1.1.1, 보조 : 1.0.0.1

  ```bash
$ cat /etc/resolv.conf
# Generated by NetworkManager
nameserver 164.124.101.2
nameserver 168.126.63.1
  ```

- `/etc/sysconfig/network-scripts/ifcfg-eth0` : 인터페이스별 네트워크 정보 조회
  - DEVICE=eth0 : 설정할 네트워크 인터페이스 장치명이 eth0 임을 지정. (파일명의 표기와 맞출 것)
  - BOOTPROTO=static : 고정 IP(static) 또는 유동IP(dhcp) 사용여부를 설정
  - BROADCAST=101.102.103.127 : 해당 IP가 속한 네트워크의 브로드캐스트 IP주소
  - HWADDR=A4:BA:DB:55:BB:FF : 네트워크 인터페이스 장치의 MAC 주소 설정 (대부분 자동 설정)
  - IPADDR=101.102.103.75 : 네트워크 인터페이스 장치의 IP 주소를 설정
  - NETMASK=255.255.255.128 : 서브넷마스크 IP 주소를 설정
  - NETWORK=101.102.103.0 : 네트워크 주소를 설정
  - GATEWAY=101.102.103.1 : 인터페이스의 게이트웨이를 설정 (이 항목을 설정하지 않으면, /etc/sysconfig/network 파일에 지정된 게이트웨이 정보가 적용)
  - ONBOOT=yes : 시스템 부팅시에 이 설정의 활성화 또는 비활성화를 선택

## 네트워크 설정 적용

```bash
$ /etc/rc.d/init.d/network restart
또는
$ service network restart
또는
$ systemctl restart network 
```
