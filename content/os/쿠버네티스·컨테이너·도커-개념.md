## 쿠버네티스 / 컨테이너 / 도커 개념

| 용어 | 역할 |
| --- | --- |
| **컨테이너** | 앱+실행환경을 묶어 어디서든 동일 실행 |
| **컨테이너 런타임** | 이미지 pull/run (Docker, containerd 등) |
| **도커** | 런타임(및 도구) 중 하나. K8s가 도커만 다루는 건 아님 |
| **쿠버네티스** | 여러 노드에 컨테이너 **오케스트레이션** (배치·교체·설정/시크릿 주입 등) |

### 배포 환경 비교

|  | Traditional | VM | Container |
| --- | --- | --- | --- |
| OS | 물리 1대당 1 | 물리 OS + VM마다 OS | 호스트 OS 1개 공유 |
| 격리 | 약함 | VM 완전 격리 | 프로세스/namespace·cgroup 격리 |
| 무게 | — | 무거움 | 가벼움 |
| 리스크 | 앱 간섭 | VM 간 전이 낮음 | 앱→OS 문제 시 전체 영향 가능 |

```bash
# 개념 확인용 예시
docker run -d --name demo nginx
kubectl get nodes
kubectl get pods -A
```
