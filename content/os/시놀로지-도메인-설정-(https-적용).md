# DSM 외부 액세스 설정

시놀로지에서 기본 제공하는 외부 액세스 서비스를 사용하여 설정한다.

---

### Synology DDNS

- **`DSM - 제어판 - 외부 액세스 - DDNS`**
- 시놀로지에서 제공하는 도메인 등록 서비스
- 외부 IP 주소가 변경되더라도 자동으로 갱신되므로 외부 서비스 연결 시 여기서 설정된 도메인으로 연결하는 것이 가장 안정적임

### QuickConnect

- **`DSM - 제어판 - 외부 액세스 - QuickConnect`**
- 시놀로지 계정 있어야 등록이 가능함
- 각종 DSM 제공 서비스에서 QuickConnect ID만으로 접속이 가능함
- 외부 도메인 설정에 문제가 생겨 DSM에 접속이 불가하여 긴급한 경우, 시놀로지 서버를 통해 DSM에 들어갈 수 있으므로 필수 설정할 것

# 공유기 설정

NAS가 연결된 상단 공유기에 접속이 가능한 경우 DDNS 및 포트포워딩 설정한다.

---

### DDNS 설정

- 공유기의 외부 IP 주소를 공유기 업체에서 제공하는 도메인으로 등록하는 설정
- 외부 IP 주소가 변경되더라도 자동으로 갱신됨
- ipTime : **`공유기 관리자 - 고급 설정 - 특수기능 - DDNS 설정`**

### 포트포워딩 설정

- 외부 : 5000 포트 → 내부 : NAS IP의 5000 포트(DSM)로 포워딩
  예) http://nugabox.iptime.org:5000 → 공유기 → 192.168.1.10:5000
- 포트를 명시하지 않고 http/https 접속 사용 시 80/443 포트에 대해 포워딩 설정
- ipTime : **`공유기 관리자 - 고급 설정 - NAT/라우터 관리 - 포트포워드 설정`**

# 네임서버 설정

소유한 도메인의 네임서버가 설정된 호스팅 사이트에서 도메인을 등록한다.

---

1. 호스팅 사이트에서 도메인 관리 → 네임서버(서브도메인) 관리에 접속
1. 추가할 서브 도메인, 레코드 종류, 레코드 값을 입력한다.
   - IP 주소로 연결 시 'A 레코드', 도메인으로 연결 시 'CNAME 레코드'
   - 고정 IP가 아닌 이상 언제든지 변경될 수 있으므로 되도록 IP 주소에 DDNS를 연결하여 'CNAME 레코드'에 도메인으로 등록한다.
     예) www / CNAME / nugabox.synology.me
1. nslookup 테스트
   [http://5l.co.kr/ns/](http://5l.co.kr/ns/)

   - 터미널 등에서 nslookup 명령어로 서브 도메인이 잘 등록되었는지 확인한다.
     ```bash
nslookup www.nugabox.com
	Server:		164.124.101.2
	Address:	164.124.101.2#53
	
	Non-authoritative answer:
	www.nugabox.com	canonical name = nugabox.com.
	Name:	nugabox.com
	Address: 11.222.222.111
     ```

# DSM 도메인 연결

---

### DSM 웹 서비스 도메인 연결

- **`DSM - 제어판 - 로그인 포털 - DSM`**
- DSM 포트(HTTP/HTTPS) : 기본 5000/5001 포트는 변경하여도 상관 없음
  → 외부 접속 시 해당 포트로 [공유기 포트포워딩](/8ce126f35c0949e980ad867c31541a66#0230182364014f5f8292444442790760) 되어 있어야 함

  → 80/443 포트 사용 시에는 [아래](/8ce126f35c0949e980ad867c31541a66#aee7fc9cb4aa4099b39a417a3744292b)에서 설정함. (즉, 여기는 80/443과 별개로 사용하는 포트임)
- '자동으로 HTTP 연결을 HTTPS로 리디렉션'에는 체크 해제
  → 체크할 경우 직접 http로 연결하는 경우 자동 https로 리디렉션 되지만, 최신 브라우저에서는 기본 https로 연결되며, 인증서 등의 문제가 발생한 경우 http로 들어가야만 하는 상황도 일어날 수 있음
- 사용자 지정된 도메인 : 설정할 도메인 입력
  → DSM의 기본 도메인으로 설정됨.

  → File Station의 공유 링크 생성 시에도 여기서 설정한 도메인으로 생성됨.

### 포트 번호 없이 DSM 접속

- **`DSM - 제어판 - 외부 액세스 - 고급`**
- 호스트 이름 또는 정적 IP : 설정할 도메인 입력
  → 위에서 설정한 DSM의 기본 포트 이외에 80/443포트를 열고 도메인과 연결함

  → 공유기에서도 80/443 포트에 대한 [포트포워딩](/8ce126f35c0949e980ad867c31541a66#877c2d41f7554be8ba2e6e4d26493715)이 되어 있어야 함

### DSM의 응용 프로그램에 도메인 연결

- **`DSM - 제어판 - 로그인 포털 - 응용 프로그램`**
- 패키지 센터에서 설치한 응용 프로그램별 로그인 포털에 대해 연결 설정 가능
- 각 응용 프로그램 '편집' → 사용자 지정된 도메인에 도메인 입력
  예) Audio Station → music.nugabox.com
- 도메인이 등록된 응용 프로그램은 **`DSM - 제어판 - 보안 - 인증서 - 설정`** 에서 서비스별로 인증서를 지정할 수 있음

# https 설정

최신 브라우저에서는 https로 기본 연결되므로 DSM 도메인 연결 시 반드시 설정한다.

---

- Let's Encrypt에서 제공하는 인증서를 시놀로지에서 쉽게 발급받아 설정할 수 있다.
- **와일드카드 인증서(\*.nugabox.com) 관련 내용은 아래 페이지 참고**


### 공유기 설정

- DSM의 https 연결에 사용하고자 하는 포트 번호로 공유기에서도 포트포워딩 한다.
- 포트 번호 없이 https 연결을 사용할 경우 443 포트를 포워딩 한다.

### 네임서버 설정

- 위 네임서버 설정과 동일하게 적용한다.
- 인증서 신청 시 입력한 서브도메인이 네임서버에 등록되어 있지 않으면 계속 실패함. 사용할 도메인을 반드시 네임서버에 모두 등록한다.

### 방화벽 설정

- **`DSM - 제어판 - 보안 - 방화벽`**
- 인증서 관련 작업 중에는 해외 IP 접속이 가능해야 하므로 방화벽 정책 풀거나 비활성화 해야 함
- 인증서 추가 또는 갱신 작업 후에 다시 활성화

### 인증서 신청

- **`DSM - 제어판 - 보안 - 인증서 - 추가`**
1. 새 인증서 추가 (이미 추가했거나 갱신할 때는 ‘기존 인증서 교체’)
1. Let’s Encrypt에서 인증서 얻기, 기본 인증서로 설정 체크
1. 인증서 얻기 정보 입력
   - 도메인 이름 : 도메인 주소 입력. 실패 시 현재 DSM 주소 넣어볼 것
   - 이메일 : 인증서에 등록할 이메일 입력. 따로 인증이 오는 건 아니지만 횟수 제한 있음
   - 주제 대체 이름 : 사용할 서브도메인 모두 포함하여 기입할 것
     예) cloud.xxxx.com; music.xxxx.com; file.xxxx.com;

     - 네임서버에 미리 등록한 서브도메인만 가능함. 아직 등록하지 않는 유효하지 않은 주소가 포함되어 있으면 실패함
     - 등록한 서브도메인은 서비스가 활성화 되어 있는, 접속이 가능한 주소여야 함
       예) Audio Station -> music.xxxx.com으로 접속이 가능해야 함
1. 신청 완료되면 웹 서버 다시 시작 (자동)
   - **공유기 80/443 포트**
   - **도메인 이름**
   - **주제 대체 이름 : 서브도메인 전체 접속 확인**
   - **방화벽 비활성화**

   \* 한 공유기에 2개 이상의 시놀로지를 사용하는 경우 : 인증 신청할 때 잠시만이라도 포트포워딩을 **현재 인증받을 시놀로지 주소의 443 포트**로 변경해야 함 (신청 끝나면 본래 쓰던 주소로 다시 돌려놓을 것)

### DSM 설정

1. 정상적으로 인증서 신청 완료 시 아래 주소로 테스트
   - https://도메인 주소:5001 (기본 포트 변경 없이 사용할 경우)
   - https://도메인 주소 ([포트 번호 없이 사용 설정](/8ce126f35c0949e980ad867c31541a66#aee7fc9cb4aa4099b39a417a3744292b)한 경우)
1. **`DSM - 제어판 - 로그인 포털`**
   - 포트 변경하고 싶으면 변경 (포트 변경했으면 공유기에서 포트포워딩 다시 해줄 것)
   - http에서 https로 자동 리디렉션에 체크 (편하기는 하나 http로 접속 불가능)

### 인증서 갱신

- Let’s Encrypt 는 무료 인증서인 대신 3개월에 한 번 씩 갱신해야 함
- 한 공유기에 두 대의 공유기 사용 및 각자 다른 도메인으로 다른 인증서까지 사용하고 있다면 인증받을 시놀로지 IP 주소로 5000/5001 (또는 80/443) 포트포워딩 변경한 후 차례대로 갱신할 것
- **인증서 작업 중에는 방화벽 비활성화 후 작업**
- **`DSM - 제어판 - 보안 - 인증서`**에서 인증서를 새로 다시 신청하거나 **`작업 - 인증서 갱신`**
- 신청 완료되면 ‘웹 서버 다시 시작’ (자동)
